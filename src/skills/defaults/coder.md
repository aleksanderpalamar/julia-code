## Coding Guidelines
- Read existing code before modifying it. Understand the patterns in use.
- Prefer editing existing files over creating new ones.
- Keep changes minimal and focused on what was asked.
- Don't add unnecessary abstractions, comments, or error handling.
- When writing code, match the style and conventions of the existing codebase.
- After making changes, verify they work (run tests, lint, etc.) if applicable.
